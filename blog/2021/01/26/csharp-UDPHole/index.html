Nat 穿透（UDP 打洞）

简介

最开始源于一次去用户现场出差，TeamViewer 与 向日葵 都是开通了会员的，但还是受到各种稀奇古怪的问题阻挠无法远程到家里的电脑和公司的电脑，回来后尝试做一个内网穿透工具以 Remote Desktop Connection(RDP) 方式连接作为以后远程的备选方案。
但是后期测试时发现 Remote Desktop Connection(RDP) 协议首先通过 TCP 进行第一次通讯建立连接以及输入用户名密码，验证用户凭据后重新以 TCP 进行远程通讯，UDP 仅作为辅助，所以改为 TCP 转发方式继续开发，UDP 打洞成功，或许以后或许可以在此基础上做些其他功能。

Nat 类型

  1. 全锥型(Full Cone)
    如果在 NAT 网关已经建立了一个 NAT 映射，那么任何外网的机器都可以通过这个映射来访问内网的电脑。
  2. 受限锥型(Restricted Cone)(IP受限锥型)
    如果在 NAT 网关已经建立了一个 NAT 映射，那么只有与其建立映射的 IP 才能通过 NAT 访问内网的电脑。
  3. 端口受限锥型(Port Restricted Cone)(IP + PORT受限锥型)
    在 IP 限制型锥型的基础上，再进行端口的限制。
  4. 对称型(Symmetric)
    即对 IP 和端口都有限制，只有和其建立连接的 IP 和端口向其发送数据才不会被丢弃。

运行流程

如图所示：局域网 NatA 中的机器 192.168.5.13(以下简称 ClientA) 请求与局域网 NatB 中的机器 192.168.1.100(以下简称 ClientB) 通过 UDP 打洞通信。

  1. 局域网内两台电脑运行 Client 程序，向服务器建立 Socket UDP 连接。
  2. ClientA 向服务端发出请求与 ClientB 建立连接。
  3. 服务端验证两台电脑在线后将 ClientB Socket UDP 信息发送给 ClientA。
  4. ClientA 发送登录信息到 ClientB，此时 ClientB 接收不到消息。
  5. ClientA 向服务端发送反向打洞请求，让 ClientB 向自己发送数据。
  6. 服务端将 ClientA 信息发送到 ClientB。
  7. ClientB 向 ClientA 发送登录信息后，双方建立打洞通信，关闭服务端程序后依然可以通信。
