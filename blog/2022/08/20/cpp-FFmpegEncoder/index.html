使用 FFmpeg 编码视频并推流或保存文件

简介

FFmpeg 是一个完整的跨平台解决方案，用于录制、转换和流式传输音频和视频。
在处理 FFmpeg 编码视频前需了解 FFmpeg 解码视频 与 FFmpeg 转发推流到 rtsp/rtmp，此文以编码推流为目的，当然也可以编码后储存为文件或作为其他作用。
设置解码方式：TCP 优化、软解码（多线程）、硬解码（CUDA、DXVA2、D3D11VA）
设置编码器：H.264 、 H.265
设置推流方式：RTSP 、 RTMP

注意事项

不支持的内容
  • 测试 RTMP 推流不支持 H265 编码，似乎可以重新编译 FFmpeg 来支持。
  • 测试 H265 解码不支持硬编码类型 NV12。
编码帧率

编码推流视频会根据帧率显示，常见的帧率：

  • 电影 24fps
  • 监控 25fps
  • 普通视频 30fps/60fps
推流到 RTSP / RTMP
  1. 创建 RTSP / RTMP 流需要通过 avformat_alloc_output_context2 创建 “rtsp” / “flv” 上下文。
  2. 创建 RTMP 流需要创建并初始化一个 AVIOContext 以访问 url 指示的资源。

  3. 创建 RTMP 流需要写入流标头前写入 sps pps，此处没有验证具体含义，可以使用其他方式写入，但是测试时对各类视频没有影响。

  4. 使用 avformat_write_header 写入流标头。

保存到本地文件

测试保存本地文件支持的文件格式有：mp4、flv、mov、ts、avi。

  1. 保存到本地文件首先需要判断文件路径的可用，如果文件夹为空时自动创建。
  2. 保存到本地文件需要通过 avformat_alloc_output_context2 创建 NULL 上下文即可，FFmpeg 可以通过文件路径中的后缀名自动创建类型。
  3. 创建并初始化一个 AVIOContext 以访问 url 指示的资源。
  4. 使用 avformat_write_header 写入流标头。
  5. 与推流不同，在写入本地视频的结尾，需要使用 av_write_trailer 写入流尾并释放数据，否则会对一些格式造成一些影响，例如： mp4 格式无法播放，flv 格式无法正确显示时间轴。
设置 pts

推流到流媒体根据目标类型需要有不同的设置。

  • 推流到 RTSP：time_base 默认为 90000，pts 平均以 3600 递增。
  • 推流到 RTMP：time_base 默认为 1000，pts 平均以 40 递增。

由于编码是多线程，测试时以当前编码间隔计算 pts，使用 av_rescale_q 从间隔时间戳转换到 pts，有效防止计算溢出的情况。
